The Advantages of Style

Cascading Style Sheets (CSS) is the language web page authors use to tel a browser or other user agent how to format an HTML document

-HTML5 defines the structure and content

-CSS defines the formatting and presentation

Style sheets provide precise control over how structural elements appear on a web page

A single style sheet can be used to provide styles for an entrie page or multiple pages on a site

Types of Styles:

-Inline style - adding murkup to invidual HTML elements

-Internal style sheets - sequential markup added to the 'style' element in the HTML document

-External style sheets - linking HTML documents to a .css file that contains your styles

-Style differently based on whether an HTML document is being viewed on a desktop or a mobile device

What are the advantages to using style sheets?

-More flexibility

-HTML5 no longer includes display-oriented or "presentational" elements and attributes meaning these may cease to be supported by browser in the near future

-Style sheets control every aspect of page display:

-Space between lines

-Character spacing

-Page margins

-Positioning of elements

-Apply chnges globally

-Instruct browser to control appearange

-Create dynamic pages using animations

Four Simple Steps to Styling

-Select the elements in a document that you want to style (selectors)

-Write declarations that apply to the selectors:

-Property name and value

-Font-family: Verdana

-Font-weight:Bold

-Padding:10px

-Link the style rules and markup

-The browser handlers the rest

Discovering Cascading Style Sheets

We understand "style sheets" but what's with the term cascading?

Cascading is the process browser use to determine which style to apply to an element?

-The last applied style will be the winner

-All web browsers follow the same rules which developers need to know

What CSS can do for a web page

Specify font type, size, color and effects

Set background colors and images

Control many aspects of text layout

Set margins and borders

Control list display

Define table layout and display

Control cursor display

Create transitions

Animate the values of CSS properties

Design multicolumn layouts

Use any of thousands of fonts

It's often a good idea to normalize before you stylize!

-Default styles might act differently in different browser

-Establishing a baseline to fix the ugliness of default browsers and inconsistencies is a good idea

CSS Structure and Syntax

Each style sheet is made up of style rules containing two parts

-Selector

-Declaration:

-Property

-Value

Declarations can configure a single or multiple values

H1 {color:teal;}

H1 {font-family:Arial;}

H1 {color:teal; font-family:Arial;}

Whitespace is irrelevant but each designer will have their own preerence as to how they write CSS style rules

Style sheets override the browsers default styling and your style declarations will alter the final appearance of the web page in the browser

Keep in mind that users can override your styles!

Keep in mind that punctuation is KEY in CSS

Mind your curly brackets and semicolons as mistyping will prevent styles from being applied!

Use a validation service like http://jigsaw.w3.org/css-validator to help find problems in your style sheets

Selectors

The universal selector (*) - matches any element and is often for global properties

-*{margin: 0px;}

-The most blunt selector there is, extremely powerful but not specific

The element type selector:

-Select a particular such as h1,p,a,div,ect...

-Works the same as universal but with a single element

ID selectors - select an element according to it's ID attribute

ID attributes are designed to be unique within a single HTML document providing extreme levels of precision with CSS

To use you preface the value of the ID you want to select with a # symbol

-#home-image {transform: rotate(-70deg);}

-container {background-color:orange;}

Class selectors - specify certain instances of an HTML marup element

To use you combine the HTML markup element with the global "class" attribute

-.warning {color:red;}

-h1.warning {color:red; font-weight:bolder;}

Elelments are allowed to hae multiple classes i.e (div class="myclass1 myclass2")

Selecting Multiple elements

-multiple elements can easily be selected

-Div, p, ul {...}

Selecting cgild elements

-If you want to select all the "p" elements inside a "div" element or other parent-child relationship

-Div p {...}

Selecting by attribute

-You can select an element based on the specific content of an attribute

-A[href=http://www.google.com] {...}

Selecting Pseudo-classes

Pseudo classes refer to the special state of an element

The most common pseudo class are those for the "a" element:

-Link (unvisited)

-Visited

-Hover

-Active (when the link is clicked)

A:hover{...}

Selecting Pseudo elements

Refers to a specified part of an element such as the first letter or the first line of the element

This is some text.

You can select the letter T using this selector p::first-letter {...}

Case insensitivy

For the most part CSS selectors and rules are case-insensitive

-Civ {Background-Color:GREEN;}

-DIV {background-coloR:green;}

The only exception is when selecting classes and ids which ARE case-sensitive!

Order of Precedence

A very important concept that can be frustrating if you are new to CSS

If you are struggling with rules not being applied check

-Case sensitivity for classes and ids

-Appropriate class values for elements

-Precedence

Understanding the rules for precedence is key!
