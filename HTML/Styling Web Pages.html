    • HTML and CSS

    HTML and CSS have a virtually inseparable relationship with one another to the extent that in order to learn web development you must understand both languages

    HTML defines the structure

    CSS defines the styling

    The supposed "unstyled page" doesn't really exist

    Default browser styles

    Custon authored styles

    Types of styles

    Inline style - adding markup to invidual HTML elements

    Internal style sheets - sequential markup added to the (style) element in the HTML document

    External style sheets - linking HTML documents to a .css file that contains your styles

    Style differently based on whether an HTML document is being viewed on a desktop or a mobile device

    Each style sheet is made up of style rules containing two part

    Selector

    Declaration:

    Property

    Value

    Declarations can configure a single or multiple values

    H1 {color:teal;}

    H1 {font-family:Arial;}

    H1 {color:teal; font-family: Arial;}

    Whitespace is irrelevant but each designer wil have their own preference as to how they write CSS style rules

    Style sheets override the browsers default styling and your style declarations will alter the final appearance of the web page in the browser

    Keep in mind that users can override your styles!

    Creating Inline Styles

    Inline styles use the style attribute are applied usally to a single element within a page

    Essentially passing CSS within the HTML element

    Not very efficient

    Sometimes they are useful such as with HTML based email

    Font Families

    Generic family - groups with a similar look like "Serif" or "Monospace"

    Serif fonts have small lines at the ends on some characters

    Sans-serif fonts do not have those lines

    Monospace fonts have characters all with the same width

    Font family - a specific font family like "Times New Roman" or "Arial"
