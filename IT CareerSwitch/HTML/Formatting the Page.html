    • Using Headings

    Heading elements implement six levels of document heading

    (h1) is the greatest

    (h6) is the least

    Used to briefly describe the content of the section it introduces

    May be used by agents to construct an outline for the document automatically

    Cautions

    Do not use headings to decrease font size

    Only use headings when there is semantic meaning, remember that HTML should be used to define the content not to style it

    Avoid skipping headings

    Best Practices

    Use (header) and (hgroup) elements to further structure the content

    Creating Paragraphs

    Paragraphs are the most basic element in a HTML document

    Define a paragraph of related text

    (p)This is a very short paragraph. It only has two sentences.(/p)

    Block level elements

    A paragraph can contain any amount of text, from a single character to thousands of words, but is typically used in the standard literary sense to contain one to a few sentences.

    There is no need to overuse (p); if the content is actually a list, heading, link or other specific type of content, there are many suitable elements available

    Emphasizing Text

    HTML contains four elements that are used to emphasize cretain bits of text as a way of differentiating it from its surrounding content

    Normally means a change in font style for most browsers such as bold or italics

    Screen readers will interpret this and result in a change in voice

    Elements

    (em)

    (i)

    (strong)

    (b)

    The (em) element:

    Marks text that has a stressed emphasis

    This emphasis would subtly change the meaning of the sentence

    (p)(em)Please(/em) remember to unplug the kettle at night.(p)

    Do not use for presentational purposes but only when additional emphasis is needed

    The (i) element:

    Used to simply mean Italics and was considered a presentational attribute in HTML4 and its use frowned upon

    Has been redefined in HTML5 and does have vaild uses

    Represents a span of text in an alternate voice or mood, or otherwise offset from the normal prose

    Use with:

  • Technical terms
  • Idiomatic phrases from othe languages
  • Names of books or ships
  • Other prose where it is common to italicize but no additional emphasis is required

    • (p)As we sailed into port, we spied the (i)Black Pearl(/i) moored at the dock.(/p)

    (p)She really does add that little bit of (i lang="fr")je ne sais quoi(/i).(/p)

    Its common that these two elements are confusing because they appear to produce the exact same result - text that is italicized

    The visual result is indeed the same

    The semantic result is different

    Use (em) when you really require additional emphasis placed on the content contained inside the (em) tags

    Use (i) when no emphasis is required

    The (strong) element

    The (strong) element is used to mark importance, seriousness, or urgency of particular text in comparsion with the text around it

    Most browsers will render (strong) as bolded text

    As with other emphasis markers do not use as a formatting tool but only when applicable

    (p)There are twenty different species living inside this enclosure.

    (strong)Warning! Do not feed them: they will eat your shoes.(/strong)(/p)

    The (b) element

    The bold element (b) is used to simply instuct the browser to display the inline text as bold

    The use of this element was frowned upon due to its presentational nature

    HTML5 has redefined (b)

    Identifies content that is stylistically offset from the rest of the text, but no more important in ter

    Creating Lists

    Lists are powerful tools used to group together similar elements and give site visitors an easy way to view groups of information

    Lists can be created for just about any type of content

    Lists use at least two components

    A markup element that tells the browser a list is on the way

    A markup element that tells the browser this particular items goes into a list

    Three types of lists:

  • Ordered lists
  • Unordered lists
  • Definition lists

    • Another name for an ordered list would be a "numbered list" because the browser will by default assign numeric values to items in the list

    The (ol) element represents a list where the items have been intentionally orederd and where changing the order of the list would change the meaning

    (ol) represents the name and beginning of a list

    (li)List item(/li)

    (li)List item(/li)

    Another name for an unordered list would be a "bulleted list"

    The (ul) element represents a list where the items have no particular order or importance

    Another type of list is the definion list that includes various name-value groups

    Consists of definition term (dt) followed by definitions of terms (dd)

    Used for:

  • Terms and definitions
  • Metadata topics and values
  • Questions and answers
  • Any other group of name-value data

    • Working With Tables

    Tables got a bad name in previous versions of HTML as they were used to control the presentation of data... not a job of the markup language

    Tables still make sense in many situation

    Lots of numbers

    Numerous fields of text that naturally fall into rows and columns

    There is a lot of markup that occurs in order to create and work with tables

    (table) -denotes the start and end of a table

    Contains a border attribute which acts as a toggle to turn the border on/off

    (tr) -denotes a table row a basic building block of HTML tables

    (td) -provides the data for the rows which ultimately defines the columns as well

    (thead) -defines the header for the table complete with (tr) and (td)

    (tbody) -defines the body of the table, a table can have multiple body sections

    (tfooter) -defines the footer

    (caption) -only one supported must come directly following (table)

    (colgroup) -only one supported must come directly following (caption)

    You can spice things up a bit...

    Use (colgroup) and styles to change the styles of invidual columns

    Use the colspan attribute to create (th) or (td) elements that span multiple columns

    Use CSS to control all presentational elements of the table such as borders, placement, shadows ect...
