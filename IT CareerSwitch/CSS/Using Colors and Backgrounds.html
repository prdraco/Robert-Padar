Working With Colors

CSS allows authors to define colors in two separate ways:

-Name

-Number:

-RGB notation

-Hex notation

There are 16 basic colors that are a part of the CSS3 specification along with a much longer list of extended colors which are supported by today's browser

When using numbers eihter RGB or HEX notation can be used and in both cases you are choosing the appropriate mixture of Red, Green and Blue (RGB):

RGB notation:

-The first number is the amount of red, the second the amound of green and the third the amount of blue

-RGB (0,255,0) - pure Green or RGB (0,0,255) - pure blue

-The actual code would be {color: rgb(0,255,0);}

Use hexadecimal notation - uses 6 digits to represent color but is the same where the first 2 digits are for red and so on:

-00 00 00

-FF 00 00

-FF 00 FF

-The actual code would be {color: #00FFFF;}

Use www.colorschemer.com/online.html

Colors can be defined using two CSS properties:

Color:

-Defines front color

-Also used to define colors for links in various states using pseudo classes

Background-Color or Background:

-Defines the color of background for an entrie page or for particular elements on the page

Working with Backgrounds

There are two option for configuring the background of an element in CSS:

-Background-color

-Background-image

The use of the background-color property is similar to how you specify the colr for a border in CSS boxes:

-Use the color name

-Use rgb notation

-Use hexadecimal notation

In order to spice things up a bit some designers utilize the background-image property instead of a standard color

CSS provides great flexibility with how we want the image to be displayed

The first step is to specify the URL to the image using the background-image property:

-Background-image:url("imagefile.jpg");

By default the image is placed in the top-left corner of the element and will automatically repeat itself bith vertically and horizontally

If this is not desirable you can the background-repeat property to modify the default setting:

-Repeat - the default setting repeats horizontally and vertically

-Repeat-x - the image will only repeat horizontally

-Repeat-y - the image will only repeat vertically

-No-repeat - the image will not be repeated

Examples:

-Background-repeat: repeat-x;

-Background-repeat: no-repeat;

By default the background image will scroll with the page but this can also be configured using the background-attachment property:

-Fixed

-Scroll

The background-position property is very useful if you want to specify which area of the background image you would like to display

-This is useful when you have an image that is larger than the element for which it is serving as a background

-If you have an image that is 200x300 and an element that is 100x100 then only a portion of the image (the top left portion) will display

-In order to display a different portion of the image you would need to change the position of the image

The most common way is to specify the background-position value using pixels and the syntax background-position:xpos ypos

The first number shifts horizontally:

-Negative values shift the image to the left

-Positive values shift the image to the right

The second number shifts vertically:

-Negative values shift the image up

-Positive values shift the image down
