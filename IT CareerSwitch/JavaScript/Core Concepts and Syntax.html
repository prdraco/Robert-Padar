    Working with Loops

    Very quicly when working with progrmming you relize that there is code that you want to be repeated

    Its more efficient to tell the browser to repeat somthing by typing it once then to have to retype countless time

    Looping isn't the difficult part

  • For every link in the page {//do this code}
  • For every field in a form on the page {//do this code}
  • 200 times {//do this code}

    • Instead the difficulty is when to stop, who's keeping track?

    One very common loop is very similar to an if statement

    var a = 1;

    if (a < 10) {

    console.log(a)

    }

    Just replace the keyword if with while and if will loop a condition evaluates to true, once it evaluates to false the loop is finished

    var a = 1;

    while (a < 10) {

    console.log(a)

    }

    Another defference with a loop is that we must have a tracking mechanism to ensure the loop ends when it should and doesn't run infinitely

    This is generally achieved by incrementing the variable

    var a = 1;

    while (a < 10) {

    console.log(a)

    a++;

    }

    The For loop is probably the most common type of loop to implement in large part because of the limited amount of code required for the loop

    for (i = 0; i < 5; i++) {

    text +="The number is " + i + "(br)";

    }

    Regardless of the type of loop there are common parts

  • Set up the index
  • Check the condition
  • Increment the condition

    • Normally a loop will continue until the condition of the oop no longer evaluates to True but there are two exceptions using JavaScript keywords

    Break

    A special word that jumps us out of the loop for for good

    Not used as often with For and While loops but you might encounter it

    Continue

    Example on "Other Program.html"

    A special word that jumps back to the top of the loop

    So it stops that particulat iteration but not the loop as a whole.
